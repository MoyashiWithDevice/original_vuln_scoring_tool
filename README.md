# 独自評価手法（Markdown整形版）

CVSSから抜粋した評価項目に、必要と感じた追加項目3つを組み合わせた新手法によって評価を行う。

---

## 評価項目

```
{
  AC：攻撃条件の難易度（3段階）
  PR：必要な権限レベル（3段階）
  UI：ユーザー操作の必要性（3段階）
  C, I, A：機密性・完全性・可用性への影響（各3段階）
  複：複合影響度（3段階）
  サ：サービス影響顕在度（3段階）
  法：法的リスク保有度（3段階）
}
```

---

## 重み付け（ポイント）

Low / Medium / High の順でポイントを記載。

| 項目 | Low | Medium | High |
|------|-----|--------|-------|
| AC | 3 | 2 | 1 |
| PR | 3 | 2 | 1 |
| UI | 3 | 2 | 1 |
| C | 1 | 3 | 5 |
| I | 1 | 3 | 5 |
| A | 1 | 3 | 5 |
| 複 | 1 | 3 | 5 |
| サ | 1 | 3 | 5 |
| 法 | 0 | 2 | 4 |

### 補足
- AC / PR / UI は「被害に直結しない」ため **最大値の引き上げを避ける意図で 3,2,1 を採用**  
- サービス影響顕在度（サ）は **サービス停止時の混乱を考慮し 1,3,5**

---

## スコア表記方法

算出した以下の2つを使用して指標とする。

- **スコアの最大値**
- **スコアの合計値**

表記形式：

```
（スコアの最大値）：（スコアの合計値）
例： 5 : 18
```

---

## 最終判定（Critical / High / Medium / Low）

最終的なランクは **scoringMatrix.xlsx** の基準に従って分類する。

---

## スコア解釈

- **スコア最大値**  
  → インパクトの大きさ（最悪の影響度）を把握できる。

- **スコア合計値**  
  → 攻撃の実行しやすさ、法的リスク、他の攻撃への波及性などの総合的評価に用いる。

この2軸により、従来よりも直感的かつ包括的な脆弱性評価が可能となる。
